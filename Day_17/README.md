# Day 17 – MVC Architecture in Backend Development

Today, I learned how to organize backend applications using the MVC (Model–View–Controller) architecture. I explored how separating the application into models, controllers, and routes helps keep the code structured and easier to maintain.

I practiced implementing this architecture in a Node.js and Express application while connecting it with MongoDB for data storage. This structure allowed me to separate database logic, application logic, and API routes into different files for better organization.
