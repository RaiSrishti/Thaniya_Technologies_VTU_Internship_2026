# Backend Development – Day 17 Notes

## What is MVC Architecture?

MVC stands for:

Model – Handles database structure and interactions  
View – Represents the user interface (usually frontend)  
Controller – Contains application logic  

## MVC Folder Structure

project-folder
│
├── models
│   └── userModel.js
├── controllers
│   └── userController.js
├── routes
│   └── userRoutes.js
└── server.js

## Advantages of MVC

- Clean and organized code
- Separation of responsibilities
- Easier debugging
- Scalable for large applications

## Learning Outcome

Today I learned how MVC architecture helps structure backend projects in a professional way by separating database operations, application logic, and route handling.
