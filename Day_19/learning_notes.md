# Backend Development – Day 19 Notes

## Importance of Error Handling

Error handling ensures that backend systems respond gracefully when problems occur instead of crashing.

## Types of Errors

Client Errors (4xx)
Example: Invalid input data

Server Errors (5xx)
Example: Database failure

## Common HTTP Error Codes

400 – Bad Request  
401 – Unauthorized  
404 – Not Found  
500 – Internal Server Error  

## Debugging Techniques

- Reading server logs
- Checking API responses
- Testing endpoints using Postman
- Reviewing code for logical mistakes

## Learning Outcome

Today I learned how proper error handling improves backend stability and helps developers debug issues more effectively.
