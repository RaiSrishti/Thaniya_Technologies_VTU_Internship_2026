# Backend Development – Day 2 Notes

## Why Backend is Important
Frontend handles only the UI, but backend:
- Stores and retrieves data
- Handles authentication and validation
- Manages business logic
- Communicates with databases

## Client–Server Architecture
- User interacts with the frontend
- Frontend sends HTTP request to backend
- Backend processes the request
- Response is sent back to frontend

## Introduction to Node.js
- JavaScript runtime environment
- Runs JS outside the browser
- Fast and scalable
- Uses non-blocking I/O

## Introduction to Express.js
- Framework built on Node.js
- Simplifies server creation
- Helps in routing and API creation
- Supports middleware

## HTTP Methods Learned
- GET → Fetch data
- POST → Send data
- PUT → Update data
- DELETE → Remove data

## What is a REST API?
- Uses standard HTTP methods
- Stateless communication
- Data is usually exchanged in JSON format

## Key Takeaways
- React alone is not enough for real applications
- Backend controls data and logic
- Full-stack development connects frontend, backend, and database
