# Backend Development – Day 22 Notes

## What is API Documentation?

API documentation explains how to use backend APIs. It provides details about endpoints, request methods, parameters, and responses.

## Key Components of API Documentation

Endpoint
Example: /api/users

HTTP Methods
GET – Fetch data  
POST – Create data  
PUT – Update data  
DELETE – Remove data  

Request Parameters
Data sent to the server (query, params, body)

Response Format
JSON data returned by the server

## Example API

GET /api/users  
Returns list of users

POST /api/users  
Creates a new user

## Importance of API Documentation

- Helps frontend developers understand APIs
- Improves collaboration
- Reduces confusion and errors
- Makes integration easier

## Learning Outcome

Today I learned how to write clear and structured API documentation, which is essential for real-world backend development.
